Abstract

The Analysis vocabulary defines general concepts and properties for analyses (e.g., trade studies, driving requirements analysis, etc.). It provides a basis for specialization by domain experts.

4. Concepts

4.1. Always

Always is a Scenario whose Constraints are always satisfied.

Supertypes
  • Scenario
Relations
  • asserts : SatisfiedConstraint (range must include at least some SatisfiedConstraint)
  • denies : UnsatisfiedConstraint

4.2. Assumption

An Assumption is a Characterization that is taken to be true for the purpose of analysis.

Supertypes
  • Characterization
Subtypes
  • Fact
Relations
  • limits : CharacterizedElement

4.3. Characterization

A Characterization provides a parametric or other qualification of zero or more model elements.

Supertypes
  • base:IdentifiedElement
Subtypes
  • Assumption
  • Explanation
  • GlossaryEntry
  • Scenario
  • project:DevelopmentState
  • vim4:IndividualNominalProperty
  • vim4:IndividualOrdinalQuantity
  • vim4:IndividualUnitaryQuantity
Relations
  • characterizes : CharacterizedElement

4.4. Constraint

A Constraint expresses a condition asserted to be true in some context.

Supertypes
  • Explanation
Subtypes
  • NominalPropertyValueConstraint
  • QuantityValueConstraint
  • SatisfiedConstraint
  • UnsatisfiedConstraint
Properties
  • hasTruthValue : xsd:boolean
Relations
  • constrains : CharacterizedElement

4.5. DrivingRequirementsExplanation

A DrivingRequirementsExplanation provides the rationale for designating a set of Requirements as driving , and makes the designation via the explains property.

Supertypes
  • Explanation
Relations
  • explains : mission:Requirement

4.6. Explanation

An Explanation captures or summarizes the results of an analysis activity and relates it to one or more other model elements. It may contain narrative prose directly or provide a reference to external products.

Supertypes
  • Characterization
  • base:AggregatedElement
Subtypes
  • Constraint
  • DrivingRequirementsExplanation
  • KeyRequirementsExplanation
  • TradeStudy
Relations
  • analyzes : AnalyzedElement
  • explains : CharacterizedElement
  • validates : CharacterizedElement

4.7. Fact

A Fact is an Assumption taken because it is thought to be true.

Supertypes
  • Assumption

4.8. GlossaryEntry

GlossaryEntry is a Characterization that associates a term with a definition.

Supertypes
  • Characterization
Properties
  • hasGlossaryDefinition : xsd:string
  • hasGlossaryTerm : xsd:string

4.9. KeyRequirementsExplanation

A KeyRequirementsExplanation provides the rationale for designating a set of Requirements as key , and makes the designation via the explains property.

Supertypes
  • Explanation
Relations

Supertypes
  • Scenario
Relations
